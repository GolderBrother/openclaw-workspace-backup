## ❌ 重要错误：Telegram username 配置问题

### 错误描述
给 OpenClaw 的 `channels.telegram.accounts` 配置添加了 `username` 字段，导致 gateway 启动失败：
```
Invalid config at /root/.openclaw/openclaw.json:
- channels.telegram.accounts.xiaoma: Unrecognized key: "username"
```

### 错误原因
- 误以为需要在 accounts 配置里加 `username` 字段来匹配 @ mention
- 实际上 Telegram accounts 不支持 `username` 字段
- bindings 里用内部 accountId（`xiaoma`）就够了

### 正确配置方式
```json
{
  "channels": {
    "telegram": {
      "accounts": {
        "xiaoma": {
          "enabled": true,
          "botToken": "8618677071:AAFura35htjucGRjXn_EGMKn0k7RUTpHtf0",
          "groups": {
            "-1003585345922": {
              "requireMention": true
            }
          },
          "groupPolicy": "allowlist",  // 不是 "open"
          "streaming": "partial"
          // 注意：不要加 username 字段！
        }
      }
    }
  },
  "bindings": [
    {
      "agentId": "dev",
      "match": {
        "channel": "telegram",
        "accountId": "xiaoma"  // 内部 accountId
      }
    }
  ]
}
```

### 关键教训
1. **Telegram accounts 不支持 username 字段** - Telegram 会自动从 bot token 获取 username
2. **bindings 用内部 accountId** - 不是公开的 @ username
3. **groupPolicy 应该是 allowlist** - 不是 open（除非你要所有群都响应）
4. **修改配置前要 validate** - 不要想当然加字段

### 修复方法
用 jq 删除所有 username 字段：
```bash
cat ~/.openclaw/openclaw.json | jq 'del(.channels.telegram.accounts[].username)' > /tmp/fix.json && mv /tmp/fix.json ~/.openclaw/openclaw.json
```

### 避免
- 不要给 telegram accounts 添加 username 字段
- 不要给 skills.entries 添加 autoRun 字段
- 不要修改配置文件时想当然添加未知字段
- 遇到配置问题先查文档或测试

## ❌ 错误2：skills.entries autoRun 字段

### 错误描述
```
Invalid config at /root/.openclaw/openclaw.json:
- skills.entries.aivectormemory-autolog: Unrecognized key: "autoRun"
```

### 错误原因
给 skills.entries 里的插件配置添加了不支持的 `autoRun` 字段

### 修复方法
```bash
jq 'del(.skills.entries[].autoRun)' ~/.openclaw/openclaw.json > /tmp/fix.json && mv /tmp/fix.json ~/.openclaw/openclaw.json
```

### 避免
- skills.entries 只支持 `enabled` 字段
- 不要添加其他未知字段
