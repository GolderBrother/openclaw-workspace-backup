## 写作风格（必须遵循）

### 整体定位
- **账号名**：James AI编程
- **领域**：AI编程、技术出海、个人IP、创业
- **读者画像**：程序员、独立开发者、想做出海/副业的技术人
- **核心特征**：务实、数据说话、专业且通俗、案例驱动

### 开头模式（固定模板）

**品牌锚点：**
> 「大家好，我是James。」——每篇文章开头必须保留

**第二句必须"炸"：**
开场白之后紧跟一句高冲击力的结论/数据/反常识判断

✅ **好的开场：**
> 「大家好，我是James。90%程序员还在让 AI 补代码，1%已经在指挥 AI 军团。」

❌ **避免：**
> 「大家好，我是James。最近我研究了一下。」（太温和）

**引入方式：**
1. **结果前置式**：「我花4天搭了个东西，现在它替我干了3个人的活。」
2. **反常识式**：「独立开发者不需要运维了。至少我不需要了。」
3. **数据轰炸式**：「10万Star，200万访问，Forbes报道。这个项目我已经在用了。」
4. **权威背书式**：「Google Chrome 团队的 Addy Osmani，两天前在 O'Reilly 发了一篇长文...这篇文章让我特别有共鸣。」

### 标题风格

**黄金公式：**
`[大比例现状] + [小比例突破] + [震撼性描述]`

**常见模式：**
- 数字对比型：「90%程序员还在让 AI 补代码，1%已经在指挥 AI 军团」
- 问题+解决方案：「AdSense被判无流量？官方聊完后我总结了8条避坑」
- 个人叙事+价值点：「我创业了！从大厂离职到独立创业者的真实经历」

**爆点升级：**
- 加冲突/反常识/记忆点
- 用"替代""干掉""不需要"等有争议感的动词
- 数字必须是具体、有冲击力的

✅ **好的标题：**
> 「90%程序员还在让 AI 补代码，1%已经在指挥 AI 军团」（数字对比）

❌ **普通标题：**
> 「AI助理搭建全记录」（太平）

### 结构特征

- **编号分段清晰**：01/02/03… 或 1）2）3）
- **模块化布局（7模块）**：
  ```
  01 → [问题现状/数据引入]
  02 → [核心判断/理论框架]
  03 → [工具验证/案例展示]
  04 → [个人实战/真实体验]
  05 → [能力要求/方法论]
  06 → [共存关系/补充说明]
  07 → [行动建议/总结升华]
  ```
- **小标题简洁有力**：
  - `[名词] + [解释]`：「大多数用AI写代码的方式，已经落后了」
  - 动词开头：「先定角色」「快速体验」「拆解思路」
- **善用分隔**：三个短横线 `---` 分隔大块模块

### 语言特点（核心特征）

**数据说话（必须）：**
- 「GitHub Copilot 现在生成开发者 46%的代码」
- 「谷歌超过 30%的新代码由 AI 生成」
- 数字必须具体、有冲击力
- 引用权威机构/知名公司

**对比手法：**
- Conductor vs Orchestrator
- 90% vs 1%
- 串行 vs 并行
- 以前 vs 现在

**口语化但不随意：**
- 「这不就是我在 OpenClaw 上干的事吗？」
- 「说到底，你只要明白...」
- 「这不像 PPT」

**专业术语自然使用：**
- 保留英文：GitHub Copilot、Cursor、Claude Code、Agent、PR
- 不过度解释
- 上下文自然融入

**善用短句：**
- 「数据摆在这，AI 写代码已经不是趋势，是事实。」
- 「一年前这些东西一个都没有。现在全有了。」

**常用连接词：**
- 「但问题是...」
- 「这就像...」
- 「说白了，...」
- 「关键不是...而是...」

## 孟健文章写作框架（必学）

参考对象：孟健（AI编程领域，实战型博主）

### 核心公式

```
开场暴击（数字+结果+立场）
    ↓
概念澄清（不是...而是...）
    ↓
真实展示（角色分工+场景证明）
    ↓
低门槛证明（第三方身份+具体时间）
    ↓
对比杀（过去 vs 现在、成本对比）
    ↓
人群定位（适合+不适合）
    ↓
理念升华（不是替代，是放大）
    ↓
行动召唤（软性推销）
    ↓
留白收尾（不强求）
```

### 开场暴击策略（3秒抓住注意力）

**模式：结果前置 + 数字冲击 + 个人立场**
```
「大家好，我是James。
一个人，6个AI助理，月成本200刀。」
```

**核心技巧：**
1. **品牌锚点**：固定开场「大家好，我是James」
2. **数字冲击**：具体且有反差（1人 vs 6个）
3. **成果展示**：不用"我研究了..."，直接展示已经在跑的东西
4. **反向承诺**：「不是画饼，不是概念图，是今天已经在跑的东西」

### 对比框架杀（建立认知冲突）

**技巧1：工具 vs 一人公司**
```
「工具是你去用它。
一人公司是它来找你。」
```

**技巧2：招人成本 vs AI成本**
```
传统：实习生月薪5000，要培训、管理、沟通
AI：月成本200刀，7x24在线，不请假
差距：78万/年 vs 2100块/年（375倍）
```

**关键点：**
- 对比必须是具体数字
- 要有真实场景支撑
- 结论要震撼

### 概念澄清（建立差异化认知）

**技巧：「不是...而是...」框架**
```
很多人一听"AI助理"就觉得是ChatGPT包了层皮。
不是的。
ChatGPT是你问一句答一句的工具。
我搭的这套系统，是一个有记忆、有分工、能主动干活的团队。
```

**建立差异化的4个维度：**
1. 有记忆：记得偏好、项目进度
2. 有分工：财务归财务，开发归开发
3. 能主动：定时检查、主动告警
4. 能协作：自动流转工作

### 真实场景叙事（增加可信度）

**技巧：时间戳式叙事**
```
早上 9:00 - 小墨主动提醒会议
上午 10:30 - 内容助理发来初稿
下午 2:00 - 开发助理PR已开好
晚上 8:00 - 财务助理报表已发
凌晨 1:00 - 增长助理排名告警
```

**技巧：手机操作场景**
```
昨天下午，我在外面开会。手机上的操作：
1. 给内容助理发了一句：「...」
2. 给增长助理发了一句：「...」
3. 给财务助理发了一句：「...」
等我开完会，三份报告都躺在聊天里了。
```

**关键点：**
- 用具体时间（不是"上午"、"下午"）
- 用真实场景（外面开会）
- 展示操作简单（发一句话）
- 强调"回来就看结果"（省时间）

### 低门槛证明（降低读者恐惧）

**技巧：自证清白**
```
「搭建只花了一天，没有写一行代码。」

「我测试了两个完全不懂技术的朋友：
- 小A：自由插画师，只会用手机，连Git都没听过
- 小B：传统销售，基本不用电脑办公
结果呢？他们都花了一晚上搞定了。」
```

**关键点：**
- 用第三方身份（不是"你也能"）
- 描述具体背景（只会用手机、不用电脑）
- 具体时间（一晚上）
- 降低承诺（不是"5分钟"）

### 人群定位（建立共鸣+筛选）

**适合的人（让读者对号入座）：**
- 独立开发者/一人创业者——身兼数职，分身乏术
- 自由职业者——没有团队但需要团队的产出
- 小团队 Leader——想让3个人干出10个人的活
- 自媒体人——内容生产、数据分析、粉丝运营全靠自己

**不适合的人（反向筛选）：**
- 对AI完全没感觉、不愿意花时间调试的
- 期望"一键搞定所有事"的——AI是员工，不是魔法

### 理念升华（情感共鸣）

**技巧：「不是替代，是放大」框架**
```
很多人担心：AI会不会把人替代了？
我的答案是：不会。
AI没有替代我，它让我变成了一个更强的我。
```

**就像计算器没有替代数学家，Excel没有替代会计师。它们放大了人的能力。**

### 留白收尾（关键技巧）

```
工具就摆在那里。
用不用，是你的事。
```

**效果：**
- 不说教（"你应该用"）
- 不催促（"赶紧学"）
- 留下想象空间（读者自己决定）
- 有态度（不强求）

### 孟健标题策略

**核心公式：`[情绪化开头] + [数字对比] + [行动表达]`**

**示例：**
- 「吹爆 OpenClaw！一个人 + 6 个 AI 助理，我再也不想招人了」
- 「90%程序员还在用ChatGPT补代码，1%已经在指挥AI军团」
- 「我创业了！从大厂离职到独立创业者的真实经历」

**拆解：**
1. **情绪化开头**：「吹爆」——口语化、有态度
2. **数字对比**：「一个人 + 6个AI助理」——反常识
3. **行动表达**：「我再也不想招人了」——个人立场

| **场景化叙事** | 时间戳式、手机操作 | 第1、2、8篇 |
| **流程图视觉化** | 箭头+缩进 | 第2篇 |
| **代码示例** | 增强技术可信度 | 第2、9篇 |
| **成本表格对比** | 详细对比 | 第2、8篇 |

| **本质区别（升华）** | 情感化描述 | 第10篇 |

### 语言特点大全

| 特征 | 说明 | 示例 |
|-----|------|------|
| **数据轰炸** | 热度+投入+结果+成本+效率 | 一周10万Star、4天、80+技能、7x24 |
| **权威背书** | 媒体+KOL | Forbes、CNET、IBM、Andrej Karpathy |
| **对比结构** | 不是...而是... | 「不是ChatGPT套壳，是有记忆、有分工的团队」 |
| **真实感受** | 口语化 | 「说实话看得眼花缭乱」「这玩意儿太香了」 |
| **推荐强调** | 强烈推荐+目标人群 | 「我强烈推荐给所有做海外产品的朋友」 |
| **个人喜好** | 主观评价 | 「我觉得挺有意思」「这几个skill是我个人比较喜欢的」 |
| **强调关键词** | 真正/完整/随时/最强大 | 「真正能帮你干活」「完整的部署流程」 |
| **数字量化** | 时间/数量/成本/效率 | 7x24、3条命令、90%、5刀 |

### 稳定特征（10篇都有）

✅ **数字量化**（标题和内容）
✅ **模块化结构**（编号分段）
✅ **口语化表达**（但不随意）
✅ **数据说话**（具体且有冲击力）
✅ **对比手法**（建立认知冲突）

### 孟健写作风格核心原则

1. **开场必须炸**：结果前置+数字冲击+个人立场
2. **对比必须狠**：过去vs现在、成本对比、工具vs团队
3. **数字必须具体**：不是"很多"，是"565个"、"6个"
4. **场景必须真实**：时间戳、手机操作、第三方身份
5. **门槛必须低**："一天搞定"、"3条命令"
6. **人群必须明确**：适合+不适合（反向筛选）
7. **理念必须升华**：不是替代，是放大
8. **收尾必须留白**：不强求，用不用是你的事

## 孟健 vs James风格对比

| 维度 | 孟健风格 | James风格 |
|-----|---------|----------|
| 开场 | 结果前置，数字冲击 | 品牌锚点 + 第二句炸 |
| 对比 | 成本对比、375倍 | 90% vs 1%、Conductor vs Orchestrator |
| 概念澄清 | 不是ChatGPT套壳 | 不是工具，是团队 |
| 场景 | 时间戳式叙事 | 真实操作场景 |
| 低门槛 | 两个不懂技术的朋友 | 直接上手，无需编程 |
| 人群定位 | 适合+不适合 | 直接对号入座 |
| 理念升华 | 不是替代，是放大 | 工具不会改变生活，愿意尝试的人会 |

**共同点：**
- 数字必须具体、有冲击力
- 大量对比框架
- 真实场景证明
- 口语化但不随意
- 短句为主

**差异点：**
- 孟健：成本对比更突出（375倍）、编号从00开始
