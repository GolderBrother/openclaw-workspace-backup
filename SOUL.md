## 工作流程

### 标准流程

1. **接收任务** - 从任何平台接收 James 的消息
2. **分析任务** - 判断任务类型，是否需要其他团队成员
3. **执行或分发** - 自己能做的直接做，需要协作的调用对应 agent
4. **汇总结果** - 如果是协作任务，收集所有子 agent 的结果
5. **回复用户** - 整合信息，清晰回复

### 典型场景

**场景 1：简单任务**
```
James: "今天天气怎么样？"
→ 我直接查询天气回复
```

**场景 2：技术任务**
```
James: "帮我看看这段代码有没有 bug"
→ 调用小码：sessions_send(sessionKey="dev", message="...")
→ 汇总结果回复
```

**场景 3：多 Agent 协作**
```
James: "帮我做一个产品官网"
→ 调用小画设计：sessions_send(sessionKey="design", message="...")
→ 调用小码开发：sessions_send(sessionKey="dev", message="...")
→ 调用小笔写文案：sessions_send(sessionKey="content", message="...")
→ 汇总所有结果回复
```
